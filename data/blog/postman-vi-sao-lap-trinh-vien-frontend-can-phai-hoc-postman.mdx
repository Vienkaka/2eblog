## Postman là gì?

Postman là một công cụ phần mềm giúp lập trình viên xây dựng, kiểm tra và tương tác với các API (Application Programming Interface). Được phát triển bởi Postman, công cụ này cung cấp một giao diện đồ họa dễ sử dụng để tạo và gửi các yêu cầu HTTP (như GET, POST, PUT, DELETE) đến các endpoint API và nhận phản hồi từ chúng. Postman hỗ trợ nhiều định dạng dữ liệu như JSON và XML và cung cấp các tính năng như kiểm tra tính đúng đắn, debug, automation và chia sẻ các bộ yêu cầu API (collections) với đồng đội hoặc cộng đồng. Điều này giúp lập trình viên xây dựng, kiểm tra và tương tác với các API một cách dễ dàng và hiệu quả.

## Vì sao lập trình viên Frontend cần phải học Postman

- Giao tiếp với API: Trong quá trình phát triển ứng dụng frontend, lập trình viên thường phải làm việc với các API để truyền và nhận dữ liệu từ máy chủ. Postman cung cấp một giao diện đồ họa dễ sử dụng để tạo và gửi các yêu cầu HTTP, nhận phản hồi từ API và kiểm tra tính đúng đắn của các yêu cầu và phản hồi này.
- Kiểm tra API: Postman cho phép lập trình viên kiểm tra tính chính xác và hiệu suất của các API. Bằng cách gửi các yêu cầu khác nhau và kiểm tra phản hồi, lập trình viên có thể đảm bảo rằng các API hoạt động như mong đợi và đáp ứng đúng các yêu cầu.
- Debugging và troubleshooting: Khi gặp sự cố trong việc gọi API hoặc nhận phản hồi từ máy chủ, Postman cung cấp công cụ mạnh mẽ để debug và troubleshooting. Lập trình viên có thể xem các thông tin chi tiết về yêu cầu và phản hồi, kiểm tra headers, body, và các thông số khác để tìm ra nguyên nhân của sự cố và sửa chữa nó.
- Tích hợp và chia sẻ: Postman cho phép lập trình viên tích hợp với các dịch vụ khác như Git, Jenkins, Newman và nhiều hơn nữa để tự động hóa quy trình kiểm tra và triển khai API. Ngoài ra, lập trình viên cũng có thể chia sẻ các bộ yêu cầu (collections) của mình với đồng đội hoặc cộng đồng, giúp tăng cường hợp tác và chia sẻ kiến thức.

## Những tính năng chính của Postman

- Tạo và gửi các yêu cầu API: Postman cho phép bạn tạo và gửi các yêu cầu HTTP như GET, POST, PUT, DELETE và nhiều loại yêu cầu khác đến các endpoint API.
- Kiểm tra và debug API: Bạn có thể kiểm tra tính đúng đắn của các yêu cầu và phản hồi API bằng cách xem thông tin chi tiết như header, body, status code và thời gian phản hồi. Postman cũng cung cấp các công cụ debug để giúp bạn tìm ra và khắc phục các lỗi trong quá trình giao tiếp với API.
- Tạo và quản lý môi trường: Postman cho phép bạn tạo và quản lý các môi trường khác nhau để dễ dàng thay đổi các giá trị biến môi trường như URL, access token và các thông số khác trong các yêu cầu API.
- Automation và testing: Postman cung cấp khả năng tự động hóa các tác vụ liên quan đến API, cho phép bạn viết và chạy các bộ kiểm tra tự động để đảm bảo tính ổn định và đúng đắn của API.
- Chia sẻ và hợp tác: Bạn có thể chia sẻ bộ yêu cầu API (collections), biến môi trường và báo cáo với đồng đội hoặc cộng đồng. Postman cung cấp các tính năng hợp tác như việc nhận xét, đánh dấu và quản lý phiên bản để dễ dàng làm việc trong nhóm.
- Tài liệu hóa API: Bằng cách sử dụng Postman, bạn có thể tạo tài liệu hóa API tự động từ các yêu cầu và phản hồi hiện có, giúp tăng cường việc hiểu và sử dụng API.
- Monitor và logging: Postman cung cấp tính năng monitor để theo dõi hoạt động của các API và ghi lại thông tin quan trọng như thời gian phản hồi, lỗi và thông tin yêu cầu.

## Những phần UI chính cần quan tâm

- Request Builder: Giao diện này cho phép bạn xây dựng và tùy chỉnh các yêu cầu API. Bạn có thể chỉ định phương thức HTTP (GET, POST, PUT, DELETE, vv.), URL, header, body, tham số query và các tùy chọn khác.
- Response Viewer: Giao diện này hiển thị phản hồi từ server sau khi gửi yêu cầu. Bạn có thể xem các thông tin chi tiết như status code, header, body và thời gian phản hồi. Nó cũng cung cấp các tính năng như định dạng hiển thị, kiểm tra lỗi và xem cấu trúc dữ liệu (JSON, XML, vv.).
- Collections Sidebar: Đây là giao diện để quản lý các bộ yêu cầu API (collections). Bạn có thể tạo, sắp xếp, chỉnh sửa và xóa các bộ yêu cầu API. Nó cũng cung cấp khả năng chia sẻ và nhập/xuất các bộ yêu cầu API.
- Environment Manager: Giao diện này cho phép bạn quản lý và chuyển đổi giữa các môi trường khác nhau (ví dụ: môi trường phát triển, thử nghiệm, sản xuất). Bạn có thể định nghĩa các biến môi trường và gán giá trị cho chúng để sử dụng trong các yêu cầu API.
- Test Scripts Editor: Giao diện này cho phép bạn viết các kịch bản kiểm tra tự động bằng cách sử dụng JavaScript. Bạn có thể kiểm tra các phản hồi, xử lý dữ liệu và thực hiện các kiểm tra tùy chỉnh. Nó cung cấp một tập lệnh API mạnh mẽ để làm việc với các phản hồi và biến môi trường.
- Newman Runner: Đây là giao diện dòng lệnh trong Postman, cho phép bạn chạy các bộ yêu cầu API từ command line hoặc tích hợp trong các quy trình CI/CD. Bạn có thể chạy các bộ yêu cầu API trên nền tảng khác nhau và xem kết quả trong giao diện dòng lệnh.

Trên đây là một vài nét về Postman, phần sau mình sẽ đi từng phần tính năng chính của Postman.
